----------------------------------------------------------------
1. PURPOSE
----------------------------------------------------------------
A native Android APK that acts as a specialized browser (WebView)
opening the Pocket Option binary-options trading platform, injecting
a JavaScript trading bot script into every page, maintaining a live
WebSocket connection to a cloud control server, and functioning as a
full browser with DevTools and Chrome extension support.

5.3  BotWebViewClient.kt
-------------------------
Extends WebViewClient.

Constructor parameters:
  getScript    : () -> String?          returns cached bot JS
  onLog        : (String) -> Unit
  onPageLoaded : (() -> Unit)?          called on page finish
  onUrlChanged : ((String) -> Unit)?    called with current URL
  getExtensions: ((String) -> List<Pair<String,String>>)?
                 returns (extensionName, jsContent) per URL

onPageFinished(view, url) sequence:
  1. onPageLoaded?.invoke()             stop SwipeRefresh spinner
  2. onUrlChanged?.invoke(url)          update URL bar
  3. Inject all matching extensions via evaluateJavascript()
  4. If url contains "pocketoption.com":
       a. evaluateJavascript(buildBridgeScript())
       b. evaluateJavascript(cleanScript(getScript()))

shouldOverrideUrlLoading():
  Always returns false — allows ALL navigation (full browser mode).
  Calls onUrlChanged with the new URL.

buildBridgeScript() returns a JS IIFE that:
  Guards: if (window.__SUPREME_BRIDGE_LOADED__) return;
  Creates window.__SUPREME_BRIDGE__ = {
    connect()       opens WebSocket to window.Android.getServerUrl()
                    sends {type:"identify", client:"bot"} on open
                    reconnects after 5s on close/error
    log(msg, type)  -> window.Android.log() + _send({type:"log"})
    stats(data)     -> window.Android.stats() + _send({type:"stats"})
    trade(data)     -> window.Android.trade() + _send({type:"trade"})
    status(s)       -> window.Android.status() + _send({type:"status"})
    _send(obj)      sends JSON if WS readyState === 1
    _handleCmd(cmd, payload)
      "start"      -> window.__SUPREME_CMD__ = "start"
      "stop"       -> window.__SUPREME_CMD__ = "stop"
      "pause"      -> window.__SUPREME_CMD__ = "pause"
      "set_amount" -> window.__SUPREME_AMOUNT__ = payload.amount
    _reloadScript(script, version)
      eval(script)  hot-reload without page refresh
  }
  Calls window.__SUPREME_BRIDGE__.connect()
  setInterval: sends ping every 10 seconds

cleanScript(raw):
  Strips Tampermonkey // ==UserScript== ... // ==/UserScript== block
  Wraps in IIFE: "(function(){\n" + cleaned + "\n})();"


5.4  BotSocketClient.kt
------------------------
Manages the OkHttp WebSocket from Android to the Node server.

Constructor:
  onCommand      : (cmd: String, payload: JSONObject?) -> Unit
  onReloadScript : (script: String, version: String) -> Unit
  onLog          : (msg: String) -> Unit

OkHttpClient config:
  connectTimeout  10s
  readTimeout     0 (infinite — stays open)
  pingInterval    20s
  retryOnConnectionFailure true

Public methods:
  connect()           launches connectLoop() on Dispatchers.IO
  disconnect()        cancels coroutine scope, cancels WebSocket
  sendLog(msg, type)  sends {type:"log", msg, logType}
  sendStats(obj)      sends {type:"stats", data:obj}
  sendTrade(obj)      sends {type:"trade", data:obj}
  sendStatus(s)       sends {type:"status", status:s}
  isConnected()       returns AtomicBoolean value

connectLoop():
  Infinite while loop on Dispatchers.IO.
  If serverWsUrl is blank or "ws://localhost:3000" -> delay 10s, retry.
  Else: calls openSocket(url), then waits backoffMs.
  Backoff: starts 2000ms, doubles each failure, caps at 30000ms.

openSocket(url):
  Creates CompletableDeferred<Unit> latch.
  Creates WebSocketListener:
    onOpen    -> wsConn=ws, connected=true, sends identify message
    onMessage -> handleMessage(text)
    onClosed  -> connected=false, latch.complete(Unit)
    onFailure -> connected=false, latch.complete(Unit)
  Calls client.newWebSocket(request, listener)
  Awaits latch (suspends until connection closes)

handleMessage(raw):
  Parses JSON. Routes:
    type "command"       -> onCommand(cmd, payload)
    type "reload_script" -> onReloadScript(script, version)

Init block (heartbeat):
  Coroutine: every 15 seconds, if connected:
    sends {type:"ping", ts: System.currentTimeMillis()}


5.5  ScriptManager.kt
----------------------
Polls GitHub for bot script updates.

Constructor:
  pollIntervalMs : Long      default 300000 (5 minutes)
  onUpdate       : (script: String, version: String) -> Unit
  onLog          : (String) -> Unit
  onChecked      : (() -> Unit)?   called after each poll completes

Public properties:
  lastCheckMs         : Long (volatile) timestamp of last poll
  pollIntervalMsPublic: Long accessor

start():
  1. Load BotPrefs.cachedScript immediately -> calls onUpdate
  2. Infinite loop: fetchIfNew() -> lastCheckMs=now -> onChecked() -> delay()

forceRefresh():
  fetchIfNew(force=true) -> lastCheckMs=now -> onChecked()

fetchIfNew(force):
  HTTP GET BotPrefs.scriptUrl
  If !force and scriptVersion not empty: adds If-None-Match header
  HTTP 304 -> skip (not modified)
  HTTP 200 -> read text, get ETag/Last-Modified
  If etag != stored version OR force:
    BotPrefs.cachedScript = script
    BotPrefs.scriptVersion = etag
    onUpdate(script, etag)

5.7  ExtensionManager.kt
-------------------------
Handles install, storage, retrieval of browser extensions.
Storage root: context.filesDir/extensions/
Index file:   context.filesDir/extensions/meta.json

installFromZip(stream: InputStream): ExtensionEntry
  1. Extract ZIP to _tmp_<timestamp>/ using ZipInputStream
     PATH TRAVERSAL GUARD: check file.canonicalPath.startsWith(destDir.canonicalPath)
  2. Find manifest.json (root, or one subdirectory deep)
  3. Parse manifest: content_scripts[].matches + content_scripts[].js
     Fallback: if no content_scripts, walk all .js files, match="<all_urls>"
  4. Rename _tmp dir to extensions/<id>/
  5. Append entry to meta.json
  Returns ExtensionEntry

installFromCrx(stream: InputStream): ExtensionEntry
  Read all bytes.
  Verify magic bytes[0..3] == "Cr24"
  CRX3: zipStart = 12 + bytes[8..11].leInt32
  CRX2: zipStart = 16 + bytes[8..11].leInt32 + bytes[12..15].leInt32
  Slice ZIP portion, call installFromZip(ByteArrayInputStream(zipBytes))

getScriptsForUrl(url: String): List<Pair<String, String>>
  Reads meta.json, filters enabled extensions whose matches patterns
  match the URL, reads each script file, returns (name, content) pairs.

URL pattern matching:
  "<all_urls>"  -> always matches
  Chrome patterns converted to regex:
    *://  -> [a-z]+://
    /*    -> /.*
    *.    -> .*\.

listExtensions(): reads meta.json, returns List<ExtensionEntry>
toggleExtension(id, enabled): rewrites meta.json
deleteExtension(id): deletes directory + removes from meta.json

startStatusPolling():
  Coroutine on Dispatchers.Main, runs every 1 second:
    updateConnDot(socketClient.isConnected())
    updateCountdown()

updateConnDot(connected):
  (connDot.background as GradientDrawable).setColor(
    if connected: 0xFF00FF88  // green
    else:         0xFFFF4444  // red
  )

updateCountdown():
  remaining = pollIntervalMsPublic - (now - lastCheckMs)
  If remaining <= 0: updateTimer.text = "🔄 ..."
  Else: updateTimer.text = "🔄 M:SS"

setupSocketClient():
  BotSocketClient(
    onCommand = { cmd, payload -> runOnUiThread { executeCommand(cmd, payload) } }
    onReloadScript = { script, version ->
      currentScript = script
      BotPrefs.cachedScript = script
      BotPrefs.scriptVersion = version
      runOnUiThread { handleLog("hot-reload"); injectScript(script) }
    }
    onLog = { msg -> runOnUiThread { handleLog(msg, "system") } }
  ).connect()

setupScriptManager():
  ScriptManager(
    onUpdate = { script, version ->
      currentScript = script; BotPrefs save
      runOnUiThread {
        Check document.readyState; if "complete" -> injectScript(script)
      }
    }
    onLog = { msg -> runOnUiThread { handleLog(msg, "system") } }
  ).start()

executeCommand(cmd, payload):
  Converts command to JS string:
    "start"      -> window.__SUPREME_CMD__='start';
    "stop"       -> window.__SUPREME_CMD__='stop';
    "pause"      -> window.__SUPREME_CMD__='pause';
    "set_amount" -> window.__SUPREME_AMOUNT__=<value>;
  webView.evaluateJavascript(js, null)

injectScript(script):
  Strips Tampermonkey header block
  Wraps in IIFE
  webView.evaluateJavascript(cleaned, callback)

showPersistentNotification():
  Creates NotificationChannel "bot" (IMPORTANCE_LOW) on API >= 26
  Posts ongoing notification with title "⚡ Supreme Bot"
  Uses android.R.drawable.ic_media_play as icon

6. NODE.JS SERVER  (bot-android/server/index.js)
----------------------------------------------------------------

Runtime: Node.js
Dependencies: express, ws, cors, node-fetch@2, http
Port: process.env.PORT || 3000

State object:
  botConnected : Boolean
  botStatus    : "OFFLINE" | "IDLE" | "TRADING" | "PAUSED"
  logs[]       : { ts, msg, type }   max 500 entries
  trades[]     : { ts, ...data }     max 200 entries
  stats{}      : { wins, losses, total, winRate, balance }
  scriptVersion: String (ETag)
  scriptCache  : String (JS text)
  lastSeen     : Number (timestamp)

REST endpoints:
  GET  /api/status         full state snapshot
  GET  /api/logs?limit=N   last N log entries (max 500)
  GET  /api/trades?limit=N last N trades (max 200)
  GET  /api/stats          win/loss stats
  POST /api/command        { cmd, payload } -> forwards to bot via WS
  POST /api/script/reload  force fetch GitHub -> push to bot
  POST /api/log            { msg, logType } manual log injection

Static: serves bot-android/dashboard/ at root /

WebSocket messages received:
  { type: "identify", client: "bot" }
    Mark socket as bot. Set botConnected=true, botStatus="IDLE".
    If scriptCache exists: send {type:"reload_script", script, version}

  { type: "identify", client: "dashboard" }
    Mark as dashboard. Send {type:"init", state:{...}} with full state.

  { type: "log", msg, logType }
    pushLog(msg, logType) -> broadcast to all dashboards.

  { type: "stats", data }
    Object.assign(state.stats, data). Recalculate winRate. Broadcast.

  { type: "trade", data }
    Push to trades[]. Broadcast to dashboards.

  { type: "status", status }
    state.botStatus = status. Broadcast to dashboards.

  { type: "ping" }
    Respond with {type:"pong", ts:Date.now()}

  { type: "command" } (from dashboard)
    broadcastToBot({type:"command", cmd, payload})

onClose for bot socket:
  botSocket=null, botConnected=false, botStatus="OFFLINE"
  pushLog("Bot disconnected"), broadcast bot_disconnected

Helpers:
  pushLog(msg, type)
    Append to logs[], trim if > MAX_LOGS, broadcastToDashboard({type:"log"})
  broadcast(msg)         sends to ALL connected WebSocket clients
  broadcastToDashboard(msg)  sends to clients where _clientType=="dashboard"
  broadcastToBot(msg)    sends only to botSocket

Script auto-update loop (every 5 minutes):
  HTTP GET GITHUB_RAW with If-None-Match: state.scriptVersion
  If 304: skip
  If 200 and new ETag: update scriptCache, broadcastToBot(reload_script)

Bot watchdog (every 15 seconds):
  If botConnected and lastSeen set and (now - lastSeen > 30000):
    pushLog warning about silent bot

8. BOT SCRIPT  (candle_V12_SUPREME.js)
----------------------------------------------------------------

Tampermonkey userscript.
The APK-specific additions use optional chaining (?.) so they
are no-ops when running in Tampermonkey (no __SUPREME_BRIDGE__).

Three integration points added for the APK:

  (A) In addLog() function:
    try { W.__SUPREME_BRIDGE__?.log?.(msg, type); } catch(_) {}

  (B) After WS trade fires:
    try {
      W.__SUPREME_BRIDGE__?.trade?.({
        dir: direction, conf: ps?.spConf, ts: Date.now()
      });
    } catch(_) {}

  (C) In stats update block:
    try {
      W.__SUPREME_BRIDGE__?.stats?.({
        wins: STATS.wins, losses: STATS.losses,
        total: STATS.total, balance: STATS.balance ?? 0,
      });
    } catch(_) {}

The bot reads two global variables set by the Android bridge:
  window.__SUPREME_CMD__    "start" | "stop" | "pause"
  window.__SUPREME_AMOUNT__ trade amount (number)

11. DATA FLOWS
----------------------------------------------------------------

(A) Boot sequence:

  onCreate
    -> BotPrefs.init
    -> setupWebView (JS interface + BotWebViewClient)
    -> setupSocketClient (BotSocketClient.connect)
    -> ScriptManager.start
         -> loads BotPrefs.cachedScript immediately -> onUpdate
         -> HTTP GET GitHub (ETag cache) -> onUpdate if new
    -> webView.loadUrl(pocketoption login)
    -> BotSocketClient connectLoop
         -> openSocket(serverWsUrl)
         -> onOpen: send {type:"identify", client:"bot"}
         -> server responds with reload_script if cached
    -> webView fires onPageFinished
         -> inject bridge script
         -> inject bot script

(B) Trade fired by bot:

  JS bot: tradeWSOrig(packet) -> trade executes on Pocket Option
  JS: window.__SUPREME_BRIDGE__.trade({dir, conf, ts})
    -> window.Android.trade(json)          [JavascriptInterface]
    -> MainActivity.handleTrade(obj)
    -> BotSocketClient.sendTrade(obj)
    -> Node server receives {type:"trade"}
    -> pushes to trades[], broadcasts to dashboard
    -> dashboard shows new row in trade history table

(C) Command from dashboard to bot:

  Dashboard button click
    -> ws.send({type:"command", cmd:"start"})
    -> Node server broadcastToBot({type:"command", cmd:"start"})
    -> APK BotSocketClient.onMessage
    -> onCommand("start", null)
    -> runOnUiThread -> executeCommand("start", null)
    -> webView.evaluateJavascript("window.__SUPREME_CMD__='start';")
    -> JS bot reads __SUPREME_CMD__ on next tick -> starts trading

(D) Script hot-reload:

  POST /api/script/reload
    -> Node: fetchScript(force=true)
    -> Node: broadcastToBot({type:"reload_script", script, version})
    -> APK BotSocketClient: onReloadScript(script, version)
    -> BotPrefs.cachedScript = script
    -> injectScript(script) -> webView.evaluateJavascript(IIFE)
    [also]
    -> Bridge JS receives reload_script via its own WS connection
    -> _reloadScript() calls eval(script)

(E) Extension injection:

  webView page finishes loading
    -> BotWebViewClient.onPageFinished(url)
    -> getExtensions(url) -> ExtensionManager.getScriptsForUrl(url)
         -> reads meta.json, filters enabled + URL-matching extensions
         -> reads JS files from filesDir/extensions/<id>/<file>.js
    -> for each (name, js): evaluateJavascript(IIFE-wrapped js)

12. DEPLOYING THE SERVER
----------------------------------------------------------------

Railway (recommended free tier):
  1. Push server/ to GitHub
  2. New Railway project -> Deploy from GitHub
  3. Set environment variable: PORT=3000
  4. Optionally: SCRIPT_URL=<github raw url of bot script>
  5. Railway auto-detects Node.js, runs: node index.js
  6. Get the public URL: https://yourapp.railway.app
  7. In APK settings: set Server URL to wss://yourapp.railway.app

Render (alternative):
  Same process, free tier available.
  Use wss:// (not ws://) since Render uses HTTPS.

13. SETTING UP THE APK AFTER INSTALL

  10. Click Start in dashboard to begin trading

14. EXTENSION INSTALLATION GUIDE
----------------------------------------------------------------

From ZIP file:
  1. Tap 🧩 button in the navigation bar
  2. Tap + Add -> "Load from ZIP / CRX file"
  3. Pick a .zip file containing a Chrome extension
     (must have manifest.json with content_scripts defined)
  4. Extension appears in list with enable toggle

From Chrome Web Store:
  1. Find extension on Chrome Web Store
  2. Copy the extension ID from the URL:
     https://chrome.google.com/webstore/detail/name/EXTENSION_ID_HERE
     (32 lowercase letters)
  3. Tap 🧩 -> + Add -> "Chrome Web Store"
  4. Paste the 32-character ID -> Install
  Note: Not all extensions work in a WebView environment.
        Extensions that modify page DOM or intercept XHR work best.
        Extensions requiring browser APIs (tabs, storage) may not work.

15. DEVTOOLS USAGE
----------------------------------------------------------------

  Tap 🔧 button in the navigation bar.
  First tap: downloads and injects Eruda.js from CDN, initializes it.
  Subsequent taps: toggles show / hide.

  Eruda provides:
    Console tab  - see all JS logs, run JS expressions
    Network tab  - see all XHR / fetch requests and responses
    Elements tab - inspect DOM tree, view computed styles
    Storage tab  - view localStorage, sessionStorage, cookies
    Info tab     - page URL, user agent, screen size

  Requires internet connection (CDN load on first use).
